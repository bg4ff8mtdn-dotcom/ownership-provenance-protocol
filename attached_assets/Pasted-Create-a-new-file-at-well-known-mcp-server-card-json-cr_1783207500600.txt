Create a new file at .well-known/mcp/server-card.json (create the
.well-known and mcp folders if they don't exist) so Smithery can read your
server's metadata directly instead of trying to auto-scan through the auth
wall. This must be served at the URL
https://ownership-provenance-protocol.replit.app/.well-known/mcp/server-card.json
and must NOT require the bearer token to access (unlike every other route,
this one specifically needs to be public, since it's just descriptive
metadata, not a way to call the actual tools).

Content:

{
  "serverInfo": {
    "name": "Ownership & Provenance Protocol",
    "version": "1.0.1"
  },
  "authentication": {
    "required": true,
    "schemes": ["bearer"]
  },
  "tools": [
    {
      "name": "create_task",
      "description": "Creates a task in pending_acceptance state. Nothing is owned until explicitly accepted."
    },
    {
      "name": "accept_task",
      "description": "Explicitly accepts a task, making the calling actor its current owner. Fails if the task already has a different current owner."
    },
    {
      "name": "report_completion",
      "description": "Reports a task as complete. Requires a provenance value (observed, reviewed, or reported) — rejected if missing or invalid."
    },
    {
      "name": "handoff_task",
      "description": "Transfers task ownership, capturing a frozen snapshot of state at the moment of transfer. The recipient must call accept_task themselves before acting as owner."
    },
    {
      "name": "get_task_status",
      "description": "Returns the complete history for a task: injection, all acceptances, all completion claims with provenance, all handoffs."
    },
    {
      "name": "list_unaccepted_tasks",
      "description": "Returns tasks with no current owner."
    }
  ],
  "resources": [],
  "prompts": []
}

Confirm this path is excluded from the bearer-auth middleware (the same
way /api/healthz already is), commit, and push to main.
